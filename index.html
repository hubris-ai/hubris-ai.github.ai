Who We Are

We are part of a new upcoming startup, Hubris, formed under the umbrella of SAP Labs Entrepreneurship Program.

Our product specializes in making most critical enterprise applications autonomous. Autonomous solutions for enterprise systems enables it to perform real time system tuning to run a given workload optimally. It automates the tuning of a database/runtime and virtually reduces the dependency on an Administrator. Thus, we guarantee our customers experience most optimal application/database performance at all times, no matter what workload is being executed. The solution stack provides knob tuning and also recommends optimal plans and physical configuration based on your demands.

Need for Autonomous Solutions

Enterprise applications are provisioned using microservices or even monoliths get executed by a language runtime. These applications often persist its state on backing services or databases. With huge number of enterprise applications and databases being consumed, provisioned via microservices or even monoliths, it becomes almost inevitable to take care of real time needs of ephimeral workloads with limited number of database administrators, operators, solution architects, etc. Also it is very difficult for a human to keep abreast with the knowledge base from frequent releases of new versions for a given system.

Security and Privacy

The tuning solution reads metrics/features and knobs form the underlying system passes it to a central tuning agent, the tuning agent reverts back with new recommendations. If your database is publicly exposed, we ask you to create technical users with limited priveleges so that we only read metadata and not actual data. However, if your database is not publicly exposed, we ask you to run our code (available on github), which reads only certain system tables and knobs and make calls to our publicly available tuning agent. The code can run as a microservice, we also have docker image for the same. As the code is publicly avaialbe you can easily verify which system tables we access.

Service Offering

We host the tuner agents on known landscapes such as GCP, AWS, Azure, etc. You can directly hook in your production databases with the tuning agent, login to the tuning agent and check the recommendations. We also have provisions to directly apply the recommendations. However, if you are reluctant to share your production instances, we ask you to copy your production data to staging systems. After copying the data to staging, we ask you to get the worst executing queries from production system, using which we create testbeds to play on staging. Finally we read the same metris/features and knobs from staging systems and provide tuning recommendation. If you have hosted your applications and database in a Virtual Private Cloud (VPC), we ask you to run the feature reader in the same VPC, which reads the features from database and sends it to central publicly exposed tuning agent.

Simple three step onboarding

We provide a easy onboarding process with minimal complexity. In the first phase, you need to decide which database to hook in with the tuning service. We recommend to initially start with staging systems. In the second phase, you have to copy data from production to staging and get worst executing queries from production. In the third phase, using worst executing queries, you have to create a test bed. The feature reader then plays the test bed on your staging database and collects features. After which the features are aggregated and sent to the central tuning agent which generates the recommendation. The process of designing a test bed is fairly simple, which also gives you flexibility to tune your database as per the workload which you select. We also provide detailed documentation on how to copy your production data to staging. After completing the three phases, you can login to the tuning agent and check recommendations.

Amazon RDS

Amazon Relational Database Service is a distributed relational database service by Amazon Web Services. It is a web service running "in the cloud" designed to simplify the setup, operation, and scaling of a relational database for use in applications.

GCP Cloud SQL

Cloud SQL is a fully managed database service that makes it easy to set up, maintain, manage, and administer your relational PostgreSQL, MySQL, and SQL Server databases in the cloud. Cloud SQL offers high performance, scalability, and convenience. Hosted on Google Cloud Platform, Cloud SQL provides a database infrastructure for applications running anywhere.

SAP CP PostgreSQL

Consume an object-relational database with PostgreSQL on SAP Cloud Platform. Our solution stack very easily integrates with SAP CP PostgreSQL.

Application Tuning - JVM

How you tune your JVM affects the performance of WebLogic Server and your applications. Figure out which JVM for your system, changing to a different JVM, optimal heap size and garbage collection, specifying heap sizes, etc.

Performance Monitor

The effectiveness of an autonomous system depends on the degree the system adopts optimal configuration with high varability of worklaods. In order to measure this, we provide a gamification model. Evaluators play a game, where the payoffs of the players depends on the performance of database.
